NC-750 & Mirror — Project Summary
The Person
French backend engineer living in rural China. Time-rich, bootstrapped (no funding). Values building something meaningful over trend-chasing. Goal: actively grow a real business.

The Brand: NC-750
A self-built design system and product universe operating at the intersection of premium industrial design (Nothing, Teenage Engineering) and absolute user sovereignty.
Three core pillars:

Total data ownership — local-first, never sold, never used for training
You are never the product — no person-level tracking, ever
BYOK or no-log relay — user controls where AI requests go

Visual identity — Enclosure Design:

Machined aluminium aesthetic, panels joined by crisp seams (no drop shadows)
Safety-orange accent (hsl(18, 100%, 53%)) as the single signal color
Typography: ClashDisplay (display/numerals) + Chillax (body) + Monospace (technical labels)
Fully themeable via CSS seed variables — steel blue, gunmetal, green, pink variants all demonstrated
The 0x00 hex mark certifies zero-person-tracking on any surface it appears

Product naming architecture:

NODE-XX — software (e.g. NODE-0M = Mirror)
UNIT-XX — hardware
CORE-XX — system primitives / infrastructure

Compliance charter (ETHOS.md) is unusually rigorous — hard constraints enforced with MUST/MUST NOT language, covering data ownership, telemetry, AI processing, identity, monetization honesty, and claims discipline. The guiding principle is integrity, not absolutism — claims must be literally true, not marketing slogans.

The Product: Mirror (NODE-0M)
What it is: A private, local-first AI career reflection tool. Conducts a structured conversational interview from user-supplied context (CV, LinkedIn export, free text), then outputs two documents:

Insight — private self-knowledge dossier (strengths, weaknesses, hidden assets, personality dimensions, elevator pitch)
Profile — polished public-facing professional website, exported as a standalone offline HTML file

Privacy architecture:

No server, no account, no telemetry
BYOK (bring your own API key) — content goes straight to the user's chosen provider
Local model support (Ollama) planned
Optional no-log relay planned for Phase 2 — stateless, content-free, backed by DPA

Current state: Functionally complete, production-quality UI. In polishing phase before first public release. Naming in transition from "Persona" → "Mirror" — needs to be applied consistently across all surfaces before launch.

Business Model
Phase 1 — Launch (current):

Free, BYOK-only
Strips all Pro references and gating code before shipping
Target: technically literate early adopters willing to set up an API key
Validates core value proposition before adding complexity

Phase 2 — Paid tier:

Subscription model
Every 3 months: in-app notification asking for feedback and reminding users to cancel if they no longer need the product (C5.2 compliance, also a genuine trust/marketing asset)
Planned paid features:

Public profile live editor (natural extension of existing static HTML output)
Job interview preparation (grounded in user's own Insight document — personalized, not generic)



Payment processor: LemonSqueezy was the plan, but post-Stripe acquisition reliability concerns (account freezes, payout issues in 2025) make it worth reconsidering. Creem identified as the strongest current alternative — MoR, EU VAT handled automatically, 0% fees up to €1K, built for indie/AI builders. Paddle is a fallback.
Pricing (Phase 1): Free. One-time purchase considered for Pro but leaning subscription for paid features.

Distribution — Open Question

Key insight on BYOK concern: Users who would "just recreate it in ChatGPT" are not the market. The actual users — people in a vulnerable career moment who want something trustworthy, private, and that just works — will not do that. The free BYOK launch self-selects for motivated early adopters who give real feedback.
